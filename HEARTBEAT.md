# HEARTBEAT.md - 心跳任务配置

## 概述
Heartbeat 是 OpenClaw 的周期性检查机制。每次心跳触发时，会执行以下检查任务。

---

## 检查时间安排

| 时间 | 检查类型 | 说明 |
|------|---------|------|
| （无） | - | - |

**说明**：以下任务已迁移到 cron 执行，更精确：
- 早上系统状态检查 → cron（周一至周五 08:00）
- 下午上下文消耗检查 → cron（周一至周五 14:00）
- 傍晚工作记录更新 → cron（周一至周五 18:00）
- 每周学习晋升检查 → cron（周日 20:00）

---

## 状态记录

通过 `memory/heartbeat-state.json` 追踪检查时间：

{
  "lastChecks": {
    "proactive": null
  }
}

## 汇报模式

### 模式一：执行任务模式

用户交代了具体任务时触发此模式。期间汇报任务执行情况，间隔按任务周期分：

| 任务规模 | 预计时长 | 汇报间隔 |
|---------|---------|---------|
| 短时任务 | < 10 分钟 | 完成后汇报一次 |
| 中时任务 | 10 分钟 ~ 2 小时 | 每 10 分钟汇报 |
| 长时任务 | > 2 小时 | 每 30 分钟汇报 |

汇报内容包含：当前步骤、进度百分比、遇到的问题（若有）。

### 模式二：静默模式

无任务、等待指令时触发此模式。状态每 **4 小时**汇报一次，内容精简：

```
🔔 状态 HH:MM
• 网关: ✅/❌
• 磁盘: XX% · 内存: XXG 可用
• [异常项若有]
```

### 模式切换

- 用户发来新任务 → 自动切换到「执行任务模式」
- 用户超过 2 小时无新消息 → 切换到「静默模式」
- 深夜（23:00-08:00）→ 不主动打扰，静默模式也不发消息

## 注意事项

- Heartbeat 触发时间近似，不保证精确
- 检查要简洁，不要输出太多内容
- 异常情况才主动通知，正常情况回复 HEARTBEAT_OK
- 深夜（23:00-08:00）不主动打扰
