# OpenClaw 核心概念 · Memory（记忆系统）

## 分层记忆架构

```
┌─────────────────────────────────────────┐
│  LAYER 1: Long-term (MEMORY.md)         │
│  长期记忆，跨会话持久化                   │
│  手动写入，自动召回                      │
└─────────────────────────────────────────┘
                    ↑
┌─────────────────────────────────────────┐
│  LAYER 2: Short-term (memory/YYYY-MM-DD)│
│  每日笔记，按日期归档                    │
│  自动创建，人工维护                      │
└─────────────────────────────────────────┘
                    ↑
┌─────────────────────────────────────────┐
│  LAYER 3: Session (对话上下文)           │
│  本次会话的对话历史                      │
│  自动管理，随会话结束清除                │
└─────────────────────────────────────────┘
```

## 记忆文件规范

### MEMORY.md（长期记忆）

位置：`~/.openclaw/workspace/MEMORY.md`

格式：
```markdown
# MEMORY.md - 长期记忆

## 基本信息
- 创建日期：YYYY-MM-DD
- 最后更新：YYYY-MM-DD

## 重要决策
- YYYY-MM-DD：[决策内容]

## 已知问题
- 问题描述及解决方案

## 偏好设置
- [用户偏好]
```

### 每日笔记

位置：`~/.openclaw/workspace/memory/YYYY-MM-DD.md`

格式：
```markdown
# memory/YYYY-MM-DD.md

## 今日完成
- [完成事项]

## 遇到的问题
- [问题及解决]

## 明日计划
- [计划事项]
```

## 记忆召回机制

```python
# 召回流程
用户问题
  → 语义搜索 MEMORY.md（关键词匹配）
  → 搜索 memory/*.md（日期相关）
  → 搜索 session history（当前会话）
  → 注入上下文
```

## 最佳实践

1. **重要决策立即记录**：不要相信"我会记住"
2. **每日工作日志**：减少回顾时间
3. **定期整理**：每周清理过时信息
4. **关键词标签**：便于语义搜索命中
