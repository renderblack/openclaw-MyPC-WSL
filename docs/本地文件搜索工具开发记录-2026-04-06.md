## 开发过程

### 1. 需求分析
- 用户需要一个本地文件搜索工具
- 可以快速查找文件名和文件内容
- 搜索范围：G盘（动画组共享目录）
- 测试案例：搜索"中铁二十四局"相关文件

### 2. 方案选择
提供5个方案供用户选择：
1. PowerShell Select-String（原生）- 慢，无模糊匹配
2. ripgrep - 快，需安装
3. Everything - 极速，只能搜文件名
4. LLM辅助搜索 - 智能但有API延迟
5. 混合方案（推荐⭐）- ripgrep + Everything

用户选择了**方案5：混合方案**

### 4. 遇到的问题

#### 问题1：PowerShell 编码问题
- **现象**：脚本中的中文注释导致解析错误
- **原因**：PowerShell 对非ASCII字符处理有问题
- **解决**：改用纯英文注释

#### 问题2：es.exe 在 PowerShell 中挂起
- **现象**：直接调用 `es.exe` 时进程挂起无响应
- **排查**：
  - Everything 服务正在运行 ✅
  - 命名管道存在：`\\.\pipe\Everything Service` ✅
  - PowerShell IPC 连接测试：成功 ✅
  - 但 es.exe 命令行调用挂起
- **解决**：使用 `cmd /c` 包装 es.exe 调用
  ```powershell
  # 挂起
  es.exe "test"
  # 正常
  cmd /c "es.exe test"
  ```

### 5. 最终脚本
```powershell
# hybrid-search.ps1
# 用法：
#   .\hybrid-search.ps1 -Keyword "关键词" [-Type content|name|all] [-Dir "目录"]

# 类型：
#   - content: 只搜文件内容 (ripgrep)
#   - name:    只搜文件名 (Everything)
#   - all:     两者都搜 (默认)
```

## 经验总结

### PowerShell 常见坑点
1. **短横线被解析成运算符**：使用 `&` 调用
   ```powershell
   # 错误
   "prog.exe" -arg value
   # 正确
   & "prog.exe" -arg value
   ```

2. **中文/特殊字符编码**：避免在脚本中使用中文注释

3. **es.exe IPC 挂起**：使用 `cmd /c` 包装

4. **exec 长时间运行被 SIGKILL**：设置合理的 timeout
