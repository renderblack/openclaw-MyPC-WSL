## 根本原因分析

### 1. 两个进程竞争

`hostname:XIONG` (大写) 和 `hostname:xiong` (小写) 同时出现，说明：
- **进程 A** (XIONG): 第一个尝试启动的进程
- **进程 B** (xiong): 接管并成功运行的进程

### 2. LogonTrigger 的问题

Scheduled Task 配置：
- 触发器: **LogonTrigger** (用户登录时启动)
- 状态: Running
- LastTaskResult: **267009**

Windows 登录过程有多个阶段：
1. Winlogon 加载用户配置文件
2. Explorer.exe 启动
3. 启动项/计划任务执行

**如果在 Explorer.exe 完全加载之前，计划任务就触发了**，可能导致：
- 环境变量未就绪
- 资源被占用
- 依赖服务未启动

### 3. Task Scheduler 超时机制

Task Scheduler 等待进程启动有超时限制（通常 30 秒）。如果超时：
- 进程被标记为失败
- 返回 267009 错误码
- **但进程可能仍在后台继续运行**

### 4. 实际发生的事

1. 用户登录 → LogonTrigger 触发
2. 进程 A 抢到资源，但启动太慢
3. Task Scheduler 等了 30 秒，标记 267009
4. 进程 B 接管并成功运行
5. Gateway 最终正常工作

## 修复方案

### 方案 1: 改用 BootTrigger（推荐）

```powershell
# 删除 LogonTrigger，添加 BootTrigger
# 不依赖用户登录，开机就启动
```

优点：
- 不依赖用户登录
- 环境变量等资源就绪
- 避免竞争条件

### 方案 2: 增加 RestartInterval

```powershell
# 设置任务失败后每分钟重试
RestartInterval: PT1M
RestartCount: 3
```

### 方案 3: 延迟启动

```powershell
# 在任务动作中添加延迟
Start /w ping -n 30 127.0.0.1 >nul
```
