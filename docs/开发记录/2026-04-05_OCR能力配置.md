### 5. 失败原因分析

#### PaddleOCR 问题
- PaddlePaddle 2.6.2 与 PaddleOCR 3.4.0 存在 API 兼容性问题
- `set_optimization_level` → `tensorrt_optimization_level` API 变更未同步
- **建议**: 降级到 PaddlePaddle 2.5.x 或等待修复

#### EasyOCR 问题
- 模型文件约 77MB，通过网络下载
- 下载中途因网络问题中断
- **建议**: 使用代理或手动下载模型

#### Windows.Media.Ocr 问题
- Windows 10 内置 OCR，但需要安装语言包
- 可能需要管理员权限
- **建议**: 通过「设置 → 时间和语言 → 语言」安装中文语言包

#### Tesseract 问题
- `pytesseract` 只是 Python 封装
- 需要单独安装 Tesseract OCR 引擎
- **建议**: 下载 Windows 版 Tesseract 安装

### 6. 已验证可行的替代方案

#### Umi-OCR（推荐）
- 你系统上已安装: `D:\Program Files\Umi-OCR_Paddle_v2.1.4\Umi-OCR.exe`

## 经验总结

### 1. OCR 工具选择原则
- **扫描件**: Tesseract 或 Windows.Media.Ocr
- **中文文档**: PaddleOCR 或 EasyOCR
- **快速处理**: Umi-OCR（已安装可直接用）

### 2. Python 环境注意事项
- 新技术栈优先选择 Python 3.10+（生态更好）
- 升级 Python 后记得更新 pip 和所有依赖
- 深度学习框架注意 CUDA 版本兼容性

### 3. 网络相关问题
- 大模型下载建议使用代理
- 国内镜像源可能不完整
- 备选方案：手动下载 + 本地安装

### 4. Windows 系统 OCR
- Windows.Media.Ocr 需要安装对应语言包
- Tesseract 需要单独下载安装包
- 权限问题可能导致 API 调用失败
