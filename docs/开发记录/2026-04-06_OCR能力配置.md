- 新版本 PaddlePaddle 需要 Python 3.9+
- 安装成功：`paddlepaddle-gpu-2.4.2.post112`
- ✅ GPU tensor 测试通过

#### Step 3：安装 PaddleOCR - 踩坑
- **踩坑 1：lmdb 源码编译失败**
  - `lmdb` 从源码构建时出现 Linux 路径问题（`/home/runner/work/py-lmdb/...`）
  - 原因：py-lmdb 的 setup.py 在 Windows 上用 MSVC 编译时路径处理有 bug
  - 解决：手动下载 `lmdb-1.3.0-cp37-cp37m-win_amd64.whl` 解决了 Python 3.7 兼容性问题

- **踩坑 2：opencv-python 版本冲突**
  - `paddleocr` 依赖链拉取了 `opencv-contrib-python 4.6.0.66`
  - 与已有的 `opencv-python 4.1.2.30` 版本冲突
  - PyMuPDF 1.22.5 被降级为 1.20.2
  - `opencv-python-headless` 的引入导致 cv2 初始化报错：`AttributeError: module 'cv2' has no attribute 'gapi_wip_gst_GStreamerPipeline'`
  - 解决：卸载 headless 版本，统一 opencv-python 为 4.6.0.66

- **踩坑 3：PaddleOCR GPU 模式初始化失败**
  - 报错：`RuntimeError: The third-party dynamic library (cublas64_112.dll) that Paddle depends on is not configured correctly`
  - 原因：PaddlePaddle 2.4.2 寻找 `cublas64_112.dll`，但系统只有 `cublas64_12.dll`（CUDA 12.2 自带）
  - 解决：复制 `cublas64_12.dll` → `cublas64_112.dll` 到 `C:\Windows\System32\`
  - **教训**：PaddlePaddle 的 DLL 命名约定是 `cublas64_[CUDA_VERSION_MAJOR][CUDA_VERSION_MINOR].dll`，不是直接用原文件名

#### Step 4：初始化成功
- PaddleOCR 初始化成功
- GPU 模式正常工作（RTX 2070 被正确调用）
- 模型自动下载（PP-OCRv4 中英文模型，~17 MB）

### 6. 失败经验总结

#### 教训 1：环境确认必须放在第一位
- **问题**：04-05 的子任务在未确认环境的情况下给出了错误结论（Python 3.11.9 + WSL2）
- **实际情况**：Windows Native PowerShell + Python 3.7.4
- **原则**：**先 `python --version` + `nvcc --version`，再开始任何安装任务**

#### 教训 2：不要迷信"旧日志"或"子任务结论"
- 子任务的结论可能基于错误前提
- **必须自己重新验证每个关键信息**
- 这次能成功，正是因为重新检查了 CUDA 版本（12.2 而不是 13.0）

#### 教训 3：Windows Python 源码编译的特殊问题
- `lmdb` 在 Windows 上从源码编译会失败（MSVC + Linux 路径混用）
- **解决**：用预编译 wheel（`.whl`），注意 Python 版本（cp37 = Python 3.7）

#### 教训 4：opencv 多版本共存会冲突
- `opencv-contrib-python` 和 `opencv-python` 必须同版本
- `opencv-python-headless` 会干扰主版本
- **原则**：统一用 `opencv-python==4.6.0.66`，不混用

#### 教训 5：PaddleOCR 离线能力被低估
- 原计划用 Tesseract 作为"离线备用"
- **实际上**：PaddleOCR 模型下载后完全离线运行（无网络请求）
- Tesseract 中文质量太差（散字），作为备用没有意义
- **修正**：PaddleOCR 本身就是完整离线方案，不需要 Tesseract 备用

#### 教训 6：CUDA DLL 命名约定
- PaddlePaddle 寻找的 DLL 名 `cublas64_112.dll` = CUDA 11.2
- 系统实际有的是 `cublas64_12.dll` = CUDA 12.2
- **需要复制并重命名**，不能只加 PATH
