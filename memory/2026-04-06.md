## 下午工作 (10:00 - 进行中)

### 目标：实现主动推送到 QQ

#### 探索过程

**1. 微信推送不可行**
- 原因：个人微信只能被动接收，无法主动推送（API 限制）

**2. QQ API 测试（失败）**
- 测试端点：
  - `https://api.q.qq.com/api/oauth2/access_token` → 404
  - `https://graph.qq.com/oauth2.0/token` → 参数错误
- **发现**：QQ Bot 使用 **WebSocket 网关**，不是 REST API
- 之前测试 REST API 端点是错误的方向

**3. QQ Bot 官方文档研究**
- 文档地址：https://bot.q.qq.com/wiki/
- 关键信息：
  - 使用 WebSocket 网关接收事件
  - 使用 REST API 发送消息
  - 支持 C2C（私聊）、Group（群聊）、Guild（频道）
  - 目标格式：`qqbot:c2c:OPENID`

**4. OpenClaw 推送机制研究**
- 文档地址：https://docs.openclaw.ai/automation/cron-jobs
- **重大发现**：OpenClaw 内置 **Cron 调度器**
- 功能：
  - 持久化任务
  - 按时唤醒 Agent
  - **可推送输出到聊天频道**
- 使用方式：`--announce --channel <channel> --to <target>`

#### 关键文档摘录
> "Cron is the Gateway's built-in scheduler. It persists jobs, wakes the agent at the right time, and can deliver output back to a chat channel or webhook endpoint."

#### 结论
- **QQ Bot 可以推送**：OpenClaw 通过 WebSocket 连接 QQ Bot，支持发送消息
- **OpenClaw Cron 支持推送**：内置调度器可直接推送结果到频道
- **下一步**：研究 OpenClaw Cron 如何配置为推送至 QQ 频道
