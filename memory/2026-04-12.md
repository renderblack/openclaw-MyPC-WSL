
## 下午：关闭微信插件

### 目标
停止微信通信，避免微信报错消息打扰

### 过程（踩坑记录）

**第1次**：只删了 `plugins.entries` 里的 `openclaw-weixin`
- 结果：无效，插件还在加载

**第2次**：又删了 `plugins.allow` 里的 `openclaw-weixin`
- 结果：还是无效

**第3次**：把 `channels.openclaw-weixin.enabled` 设为 false
- 结果：仍然加载

**第4次**：移动扩展目录 `extensions/openclaw-weixin` 到 `extensions.disabled/`
- 结果：还是有错误日志

**第5次**：删除配置文件里 `channels.openclaw-weixin` 整个段落
- 结果：✅ 成功！

### 根因分析

OpenClaw 插件加载有3个层次：
1. `extensions/` 目录 — 物理文件位置（扫描加载）
2. `plugins.allow` — 允许加载的插件列表
3. `plugins.entries` — 运行时注册的插件条目
4. `channels.xxx` — channel 配置段落

只改配置不移动扩展文件，插件依然会被加载。

### 正确关闭插件步骤
1. 移动 `~/.openclaw/extensions/<插件名>` 到其他目录
2. 从 `plugins.allow` 删除
3. 从 `plugins.entries` 删除
4. 删除 `channels.<插件名>` 配置段落（如有）
5. 重启 Gateway

### 最终状态
- 微信插件已禁用
- QQ Bot 正常运行
- Gateway 无报错
